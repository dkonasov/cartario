## What I do

- find section corresponding to the user story in `docs/plan.md`
- implement tasks listed in that section
- run tests and make sure they pass
- run the project in background and open it in browser with chrome-devtools. Ensure the implemented feature works as expected and doesn't cause any issues in the app.
- run linter and formatter, fix any issues
- mark tasks related to the user story as done in `docs/plan.md` by replacing `[ ]` with `[ x ]`

## When to use me

Use me when you need to implement a user story. DON'T use me when you need to plan a user story, use `plan-user-story` skill for that.
